## Agentic RL 后训练方法调研（oc_ac5cd 群）

### 背景
- 张海林做超大规模模型后训练，目标是提升基座模型通用能力
- 纯 RL（GRPO/DAPO）和 OPD（On-Policy Distillation）已实现
- 需要调研额外可能有增益的方法

### 调研的 7 个方法（来源：青稞AI 公众号 Hao Bai 文章）
1. OPD (On-Policy Distillation) - TML 提出
2. OPSD (On-Policy Self-Distillation) - 无需 teacher
3. SDPO (Segment-Level DPO) - 多轮对话
4. POPE (Privileged On-Policy Exploration) - 难题 prefix 引导
5. InT (Intervention Training) - 步骤级修正
6. DigiRL - digital agent RL
7. Digi-Q - digital agent Q-learning

### 分析框架：The Bitter Lesson
- 用 Sutton 的 Bitter Lesson 哲学重新评估
- 核心原则：通用方法 + 计算 >> 精巧设计 + 人类知识注入
- 好方法 = scale with compute；坏方法 = human knowledge injection

### 最终结论
- **主力**：纯 RL（已实现）+ OPD（已实现）
- **值得实验的增量方法**：
  1. POPE（优先级最高）— 解决 pass@k=0 的难题探索问题，改动小
  2. OPSD（探索性）— 无 teacher 时的自蒸馏，需验证大模型上信号是否足够
- **排除**：InT（too human-knowledge-heavy）、SDPO（场景不匹配）、DigiRL/Digi-Q（完全不适用）

### Magma 优化器讨论
- 论文：arXiv:2602.15322（Google + 西北大学）
- 核心：随机掩码 50% 参数更新 + 动量-梯度对齐 → 隐式几何正则化
- 结果：1B 模型上比 Adam 降 19% 困惑度，比 Muon 降 9%
- 我的分析：有启发但不本质
  - 实验太小（最大 1B），无大规模验证
  - RL 后训练场景可能不需要（另一篇论文 2602.07729 发现 RL 阶段 SGD 就够用）
  - 2026年2月优化器论文井喷，可能是同一现象的不同视角
  - 建议等大规模复现再考虑
